# UI Component Inspiration

This document references design inspiration sources for Guidr's UI components.

## Component Libraries & Resources

### 21st.dev
- **URL:** https://21st.dev/community/components
- **Use for:** Modern component patterns, clean interfaces
- **Notes:** Good source for ready-made components and design patterns

### Lukacho UI
- **URL:** https://ui.lukacho.com/components
- **Use for:** Animated components, Tailwind + Framer Motion examples
- **Notes:** Great for micro-interactions and smooth animations

### Aceternity UI
- **URL:** https://ui.aceternity.com/components
- **Use for:** Unique, engaging component designs
- **Notes:** Creative approaches to common UI patterns

### MagicUI
- **URL:** https://magicui.design/docs/components
- **Use for:** Component library with unique animations
- **Notes:** Helpful for adding polish and delight to interactions

## Design Goals

We want to avoid the "generic AI app look" and instead create:

### Visual Feel
- **Warm and earthy** - Using our color palette (Moss, Mustard, Eggshell)
- **Engaging** - Not boring or sterile
- **Welcoming workspace** - Feels like a helpful tool, not a cold interface

### Component Types to Reference

1. **Cards**
   - Program cards for search results
   - Profile cards for dashboard
   - Recommendation cards with tier labels

2. **Navigation**
   - Sidebar navigation with active states
   - Breadcrumbs for deep navigation
   - Mobile-responsive menu

3. **Forms**
   - Profile editing forms
   - Academic record forms
   - Document upload interfaces

4. **Dashboards**
   - Welcome dashboard with action cards
   - Progress indicators
   - Status badges

5. **Modals & Dialogs**
   - Essay creation modal
   - Email draft modal
   - Confirmation dialogs

6. **Data Display**
   - Program detail pages
   - Essay review feedback
   - Recommendation tier groupings

## Implementation Notes

- Adapt inspiration to match Guidr's color palette
- Maintain consistency with our design system
- Prioritize usability over flashy animations
- Ensure all components are accessible
- Test on mobile devices
